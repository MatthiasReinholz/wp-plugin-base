# Security Policy

## Reporting A Vulnerability

Report suspected vulnerabilities in `wp-plugin-base` privately through GitHub Security Advisories when possible:

- open the `Security` tab for this repository
- choose `Report a vulnerability`
- include reproduction steps, impact, and any suggested mitigation

If GitHub Security Advisories are unavailable for your report, contact the maintainers privately before opening a public issue.

Do not publish exploit details or credential material in public issues or pull requests.

## Scope

This policy covers the foundation repository itself, including:

- managed workflow templates
- vendored scripts
- release and update automation
- dependency policy files and security documentation
